# CV Model Lab Server

Optional FastAPI backend that enables **server mode** for CV Model Lab. It
browses datasets under configured allowed roots, parses and indexes COCO
annotations/predictions server-side, serves image bytes and thumbnails, runs
COCO AP metrics and TP/FP/FN evaluation, and serves the Flutter Web/PWA build.

The server is **read-only** with respect to datasets and prediction files. The
only data it writes is its own cache and logs. It never edits annotations,
predictions, or projects.

See the full guide in [`docs/server_mode.md`](../docs/server_mode.md).

## Requirements

- Python 3.9+
- [`uv`](https://docs.astral.sh/uv/) (recommended)

## Install

```bash
cd server
uv venv
uv pip install -e ".[dev]"
```

## Configure

Copy the example config and edit the allowed roots:

```bash
cp server.example.yaml server.yaml
```

`allowed_roots` is mandatory — the server will only ever read files inside these
directories. Path traversal and symlink escapes are rejected.

## Run

```bash
# from the repo root
scripts/run_server.sh server/server.yaml
# or directly
cd server
uv run python -m cvmlab_server.main --config server.yaml
```

If `api_key` is not set, the server runs in **open access** mode. In a terminal
it asks for confirmation (default: no). When started non-interactively it
refuses to start unless you pass `--allow-unauthenticated`.

To require authentication, set `api_key` in the config. Clients must then send
the `X-CVML-API-Key` header.

## Serving the PWA

Build the Flutter web app and point `static_web.root` at it:

```bash
flutter build web
# static_web.root defaults to ../build/web
```

The PWA is then available at `http://<host>:<port>/` and talks to the same
origin's API automatically.

## Tests

```bash
cd server
uv run pytest
```

The parity tests in `tests/test_parity.py` assert that the Python evaluation
reproduces the Dart core exactly. Their fixtures are generated by
`flutter test test/server_parity/generate_fixtures_test.dart`.
