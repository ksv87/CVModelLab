# Мобильный PWA / адаптивный веб

[English version](../mobile_pwa.md)

CV Model Lab адаптирует свою сборку Web/PWA под узкие и мобильные экраны. Одно и
то же приложение работает как desktop-приложение, как браузерное приложение на
широком экране и как адаптивное мобильное веб-приложение — без добавления
нативных таргетов Android или iOS.

## Когда применяется мобильная вёрстка

Интерфейс переключается между тремя классами ширины по доступной ширине экрана:

| Класс    | Ширина (logical px) | Вёрстка |
|----------|---------------------|---------|
| Compact  | `< 700`             | Мобильная вёрстка с нижней навигацией |
| Medium   | `700 – 1099`        | Desktop-вёрстка с уменьшенными боковыми панелями |
| Expanded | `>= 1100`           | Полная desktop-вёрстка |

Таким образом «мобильный веб» охватывает Android Chrome, iOS Safari,
установленный на телефон PWA и окно desktop-браузера, суженное до узкой ширины, —
все они используют компактную вёрстку.

## Сценарий работы в браузере / PWA

- **Локальный standalone-веб** сохраняет лёгкий браузерный сценарий: открытие
  COCO annotations, предсказаний и изображений через браузерный file picker и
  режим Web/PWA restore. Просмотр изображений, метрики, сравнение и экспорт
  отчётов работают и на экране размером с телефон.
- **Удалённый (серверный) режим** подключается к self-hosted FastAPI backend и
  просматривает датасеты на стороне сервера так же, как на desktop.

## Компактная навигация

На компактной ширине трёхпанельный desktop-рабочий стол заменяется нижней
навигацией с шестью разделами:

- **Проект** — информация о проекте/сервере, выбор активного model run, пороги
  оценки, статус COCO AP и перезагрузка.
- **Изображения** — браузер изображений. Фильтры открываются в bottom sheet,
  тап по изображению открывает полноэкранный мобильный просмотрщик.
- **Метрики** — Dashboard, Dataset Health, Confusion Matrix, Worst Cases и
  Recommendations, выбираемые лентой чипов.
- **Сравнение** — открывает попарное / multi-model сравнение (нужно минимум два
  model run).
- **Отчёты** — экспорт отчётов и аннотированных изображений.
- **Ещё** — язык, тема, информация о подключении к серверу и действия с проектом.

## Мобильный просмотрщик изображений

Полноэкранный просмотрщик поддерживает:

- зум щипком и панорамирование;
- вписывание в экран;
- следующее / предыдущее изображение и следующая / предыдущая ошибка;
- bottom sheet параметров наложения (GT, предсказания, TP, FP, FN, подписи,
  score, IoU);
- bottom sheet деталей рамки (тип, класс, score, IoU, bbox, причина, model run)
  при тапе по рамке.

Отсутствующие изображения показывают читаемый плейсхолдер.

## Таблицы и фильтры на маленьких экранах

Широкие таблицы деградируют аккуратно. Список изображений использует плитки,
фильтры переезжают в bottom sheet, а Confusion Matrix по умолчанию показывает
список **Топ путаниц классов** с полной матрицей за переключателем «Top pairs»
(полная матрица при необходимости прокручивается по горизонтали). Лидерборды и
поклассовые таблицы остаются доступны и прокручиваются по горизонтали на узких
экранах.

## Отчёты в мобильном вебе

Диалог экспорта помещается на узких экранах, переносит длинные подписи,
сохраняет доступность переключателей AP и показывает уведомление, что большие
экспорты могут выполняться долго в мобильном браузере. Сама генерация отчётов не
изменилась.

## Поведение same-origin PWA

- Когда PWA **раздаётся backend-ом CV Model Lab**, приложение проверяет
  `/api/config` по своему origin. Если backend обнаружен (валидный config или
  `401`, когда сервер требует ключ), URL сервера фиксируется на этом origin и не
  редактируется; при необходимости появляется запрос API-ключа.
- Когда приложение работает **standalone** (например `flutter run -d chrome` или
  статический хостинг без backend), проверка `/api/config` не проходит, поэтому
  поле URL сервера остаётся редактируемым, а локальные сценарии через file picker
  доступны.

## Известные ограничения

- Нативные приложения Android и iOS не входят в этот этап; мобильный опыт
  обеспечивается только адаптивным Web/PWA.
- Оценка COCO AP по-прежнему не запускается в браузере; импортируйте готовый JSON
  с AP-метриками или считайте AP на desktop/сервере.
- Таблицы multi-model сравнения на очень узких экранах используют горизонтальную
  прокрутку, а не полную карточную вёрстку.
- Очень большие экспорты отчётов могут выполняться медленно в мобильном браузере.
