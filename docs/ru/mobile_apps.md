# Android и iOS как удалённые клиенты

Приложения CV Model Lab для Android и iOS работают только как удалённые
клиенты для optional FastAPI server mode. Они не открывают локальные COCO
annotations, prediction JSON, каталоги изображений или файлы локальных проектов
с устройства.

Имя native app: **CV Model Lab**. Android `applicationId` и iOS bundle
identifier: `ru.ksv87.cvmodellab`.

## Требования

- Запущенный CV Model Lab Server.
- Сетевой доступ с мобильного устройства к адресу сервера.
- API-ключ, если сервер настроен на обязательную авторизацию.

Настройка сервера описана в [Серверном режиме](server_mode.md).

## Подключение к серверу

1. Запустите мобильное приложение.
2. Стартовый экран показывает **Режим удалённого клиента**,
   **Подключиться к серверу** и **Открыть недавний удалённый проект**.
3. Нажмите **Подключиться к серверу**.
4. Введите адрес сервера вручную, например `http://192.168.1.20:8080`.
5. Введите API-ключ, если он требуется.
6. Включайте **Запомнить API-ключ на этом устройстве** только если ключ должен
   храниться в настройках приложения на этом устройстве.
7. Нажмите **Проверить подключение**, затем откройте проект сервера.

Сохранённый API-ключ можно удалить на том же экране подключения. API-ключи не
записываются в `.cvmlab.json` и в descriptors недавних удалённых проектов.

## Проекты из server manifest

Если на сервере включены manifests, после успешного подключения приложение
показывает список настроенных проектов сервера. При открытии manifest project
используются server-side пути из manifest. Повторное открытие недавнего
удалённого проекта не требует повторного выбора путей.

## Произвольные пути на сервере

Если сервер разрешает custom paths, приложение использует server file browser
для выбора:

- COCO annotations JSON;
- каталога изображений;
- одного или нескольких prediction JSON;
- optional AP metrics JSON, если это поддерживается server workflow.

File browser показывает только разрешённые server-side roots. Это не локальный
file picker устройства.

## Что отключено на мобильных

В Android и iOS приложениях намеренно отключены:

- локальные standalone dataset projects;
- выбор локальных COCO annotations, predictions и изображений с устройства;
- импорт локального project file с device storage;
- локальный Python COCO AP evaluator;
- broad Android storage permissions и iOS document-picker dataset access.

Desktop local mode и Web/PWA local mode остаются доступными на своих
платформах.

Переключатели темы и языка остаются в app bar. Отдельного пункта настроек на
стартовом мобильном экране нет.

## Отчёты и экспорт

Поддержка отчётов на мобильных ограничена в этой версии. Экспорты, которым
нужен выбор локальной папки, отключены в native mobile apps. Для полного HTML,
CSV, XLSX, PDF и annotated-image export workflow используйте Desktop или
Web/PWA.

## Troubleshooting

- **Сервер недоступен:** проверьте, что телефон и сервер находятся в одной сети
  или что URL сервера открывается из браузера телефона.
- **Неверный API-ключ:** введите ключ заново, повторите проверку подключения и
  удалите сохранённый ключ, если были сохранены устаревшие credentials.
- **Нет проектов сервера:** включите manifests в конфигурации сервера или
  используйте custom server paths, если сервер их разрешает.
- **Custom paths недоступны:** проверьте allowed roots и поддержку custom paths
  в конфигурации сервера.
