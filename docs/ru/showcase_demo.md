# Демонстрационный пример (Showcase Demo)

[English version](../showcase_demo.md) | [README.ru](../../README.ru.md)

CV Model Lab поставляется с синтетическим демонстрационным датасетом и
сохранённым project-файлом, который загружает demo assets из `demo/showcase_coco/`.

## Что такое Showcase Demo?

Демонстрационный датасет — синтетические сцены дорог, нарисованные из геометрических
фигур и градиентов. Никаких фотографий, никаких защищённых авторским правом материалов.
Каждый пиксель генерируется скриптом
`tools/generate_showcase_dataset/generate_showcase_dataset.py`.

Датасет намеренно спроектирован так, чтобы демонстрировать все ключевые возможности
CV Model Lab: истинно-положительные, ложно-положительные, ложно-отрицательные ошибки,
классовую путаницу, малые объекты и значимые различия между тремя моделями.

## Как открыть

На экране открытия проекта нажмите **Open project** и выберите
`demo/showcase_coco/showcase_coco.cvmlab.json`.

Сохранённый project загружает:
- GT-аннотации (`annotations.json`)
- Три модели с предсказаниями
- Предвычисленные COCO AP-метрики для всех трёх моделей

Альтернативно: открыть файлы из `demo/showcase_coco/` вручную:

1. Открыть **annotations.json** как аннотации.
2. Добавить первый model run: **predictions_model_a.json**.
3. Добавить ещё два запуска через *Add model run*: Model B и Model C.
4. Импортировать AP-метрики для каждого запуска через кнопку
   *Import AP metrics JSON* в панели «COCO AP Metrics».

## Содержимое датасета

| Параметр | Значение |
|----------|---------|
| Изображений | 40 синтетических PNG (960×540) |
| GT-аннотаций | 87 |
| Категорий | 5 (см. ниже) |
| Model runs | 3 (A, B, C) |
| Предсказаний (A) | 72 |
| Предсказаний (B) | 106 |
| Предсказаний (C) | 70 |

### Категории

| ID | Имя | Что изображает |
|----|-----|----------------|
| 1 | `red_light` | Красный сигнал светофора |
| 2 | `yellow_light` | Жёлтый сигнал светофора |
| 3 | `green_light` | Зелёный сигнал светофора |
| 4 | `pedestrian_sign` | Синий знак пешеходного перехода |
| 5 | `background_distractor` | Уличный фонарь / круглый знак (не светофор) |

### Типы сцен

| Изображения | Сцена | Что проверяем |
|-------------|-------|---------------|
| 1–5 | Один крупный светофор | Чистые TP-примеры |
| 6–10 | Два светофора | Multi-object detection |
| 11–15 | Светофор + знак пешехода | Классовая путаница |
| 16–22 | Уличные фонари (дистракторы) | FP / class confusion |
| 23–28 | Маленькие дальние светофоры | Recall малых объектов |
| 29–33 | Только фон (без GT) | Уровень FP на пустых сценах |
| 34–37 | Затемнённые / тёмные сцены | FN у всех моделей |
| 38–40 | Сложные смешанные сцены | Полный mix TP/FP/FN |

## Характеристики моделей

### Model A — Baseline (базовая)

- **AP**: 0.52 | **AP50**: 0.74
- Сбалансированная модель с умеренными P/R.
- Пропускает ~35% малых объектов.
- FP на дистракторах ~35% изображений.
- Редкая классовая путаница: yellow → green (~15%).

### Model B — High Recall (высокий recall)

- **AP**: 0.47 | **AP50**: 0.70
- Лучший recall: почти все GT-объекты обнаруживаются, включая малые.
- Больше всего FP: на дистракторах и пустых сценах.
- Больше классовой путаницы: red → yellow, yellow → green.
- Низкий precision снижает AP несмотря на лучший recall.

### Model C — High Precision (высокая точность)

- **AP**: 0.61 | **AP50**: 0.82
- Лучший AP: высококонфидентные предсказания, почти нет FP.
- Пропускает 60–90% малых объектов (`AP_small` = 0.19).
- Почти нет FP на пустых сценах и дистракторах.
- Редкая путаница: pedestrian sign → green light.

## Экраны для изучения

|-------|--------------------------|
| **Workspace / Dashboard** | Метрики по классам, AP-карточки, COCO AP summary |

## Примеры отчётов

Сгенерированные примеры лежат в `demo/showcase_coco/reports/`:

| Папка | Содержимое | Как использовать |
|-------|------------|------------------|
| `en/all reports/` | HTML, PDF, XLSX и CSV для full evaluation Model A | Основной portfolio sample |
| `en/pairwise compare/` | Pairwise A vs B HTML/PDF/CSV export | Пример сравнения моделей |
| `en/multi-model compare/` | A/B/C leaderboard, class ranking, disagreements, regression matrix | Пример multi-model анализа |
| `ru/multi-model compare/` | RU multi-model HTML/PDF/XLSX/CSV export | Русскоязычный пример |

RU full и pairwise папки можно использовать как технические samples, но часть table labels пока остаётся английской. CSV-заголовки намеренно остаются английскими во всех локалях.

Чтобы пересоздать отчёты:

1. Открыть `demo/showcase_coco/showcase_coco.cvmlab.json` через **Open project**.
2. Переключаться между model runs для просмотра метрик.
3. Нажать **Export Report** → выбрать HTML / CSV / XLSX / PDF и язык отчёта.
4. Открыть **Model Compare** и экспортировать pairwise или multi-model отчёты.

Все форматы отчётов включают AP-метрики, если они доступны.
Полный список файлов и validation notes: `demo/showcase_coco/reports/README.md`.

## Почему синтетические данные?

Скриншоты и демонстрационные материалы в публичных репозиториях не должны содержать
реальные изображения, которые могут быть проприетарными или приватными. CV Model Lab
проверялся на реальных внутренних object detection датасетах, но эти изображения и
отчёты не публикуются.
