# Showcase Demo

[Русская версия](ru/showcase_demo.md) | [README](../README.md)

CV Model Lab ships with a synthetic showcase dataset and a saved project file
that loads the demo assets from `demo/showcase_coco/`.

## What is the Showcase Demo?

The showcase dataset is a synthetic road-scene dataset created entirely from
geometric shapes and gradients. It contains no photographs and no copyrighted
assets. Every pixel is generated by
`tools/generate_showcase_dataset/generate_showcase_dataset.py`.

The dataset is intentionally designed to demonstrate every major feature of
CV Model Lab: true/false positives, false negatives, class confusion, small
objects, and meaningful differences between three model variants.

## Opening the demo

On the project open screen, click **Open project** and select
`demo/showcase_coco/showcase_coco.cvmlab.json`.

The saved project loads:
- Ground truth annotations (`annotations.json`)
- Three model runs with their predictions
- Precomputed COCO AP metrics for all three models

Alternatively you can open the files manually from `demo/showcase_coco/`:

1. Open **annotations.json** as annotations.
2. Add model run **predictions_model_a.json** (Model A — Baseline).
3. Add more runs via *Add model run*: Model B and Model C.
4. Import AP metrics per run via the *Import AP metrics JSON* button in the
   COCO AP Metrics dashboard panel.

## Dataset contents

| Property | Value |
|----------|-------|
| Images | 40 synthetic PNG (960×540) |
| GT annotations | 87 |
| Categories | 5 (see below) |
| Model runs | 3 (A, B, C) |
| Predictions (A) | 72 |
| Predictions (B) | 106 |
| Predictions (C) | 70 |

### Categories

| ID | Name | What it represents |
|----|------|--------------------|
| 1 | `red_light` | Red traffic light signal |
| 2 | `yellow_light` | Yellow traffic light signal |
| 3 | `green_light` | Green traffic light signal |
| 4 | `pedestrian_sign` | Blue pedestrian crossing sign |
| 5 | `background_distractor` | Street lamp / round sign (not a traffic light) |

### Scene types

| Images | Scene | What to inspect |
|--------|-------|----------------|
| 1–5 | Single large traffic light | Clean TP examples |
| 6–10 | Two traffic lights | Multi-object detection |
| 11–15 | Traffic light + pedestrian sign | Class confusion |
| 16–22 | Street lamps (background distractors) | FP/class confusion |
| 23–28 | Small distant traffic lights | Small-object recall |
| 29–33 | Background only (no GT) | False-positive rate |
| 34–37 | Obscured / dark scenes | All-model false negatives |
| 38–40 | Complex mixed scenes | Full TP/FP/FN mix |

## Model characteristics

### Model A — Baseline

- **AP**: 0.52 | **AP50**: 0.74
- Balanced model with moderate precision and recall.
- Misses ~35% of small objects.
- Has some false positives on background distractors (~35% of distractor images).
- Occasional class confusion: yellow → green (~15%).

### Model B — High Recall

- **AP**: 0.47 | **AP50**: 0.70
- Best recall of the three; catches nearly all GT objects including small ones.
- Produces the most false positives: heavy FP on background distractors and
  background-only images.
- More class confusion: red → yellow, yellow → green.
- Lower precision leads to lower AP despite best recall.

### Model C — High Precision

- **AP**: 0.61 | **AP50**: 0.82
- Best AP of the three; high-confidence predictions with very few FP.
- Misses 60–90% of small objects (`AP_small` = 0.19).
- Almost no FP on background images or distractors.
- Rare class confusion (pedestrian sign → green light).

## Screens to inspect

| Screen | What to look for |
|--------|-----------------|
| **Workspace / Dashboard** | Per-class metrics, AP cards for each model, COCO AP summary |
| **Error Browser** | FP on background images (B has most); FN on dark scenes |
| **Dataset Health** | Class imbalance (red vs yellow), small-object ratio |
| **Confusion Matrix** | yellow→green and red→yellow confusions |
| **Worst Cases** | Dark/obscured images; background-heavy images |
| **Recommendations** | Small-object gap, FP rate, class imbalance warnings |
| **Model Compare → Pairwise** | A vs B: B has many more FP; A vs C: C misses small |
| **Model Compare → Multi-model** | Leaderboard, per-class ranking, image disagreements |
| **Compare → Regression Matrix** | Which model pairs share errors |

## Sample reports

Generated examples are committed under `demo/showcase_coco/reports/`:

| Folder | Contents | Best use |
|--------|----------|----------|
| `en/all reports/` | HTML, PDF, XLSX, and CSV files for the Model A full evaluation | Main portfolio sample |
| `en/pairwise compare/` | Pairwise A vs B HTML/PDF/CSV export | Model comparison sample |
| `en/multi-model compare/` | A/B/C leaderboard, class ranking, disagreements, regression matrix | Multi-model sample |
| `ru/multi-model compare/` | Russian multi-model HTML/PDF/XLSX/CSV export | Russian-language sample |

The Russian full and pairwise folders are useful technical samples, but some table labels remain English. CSV headers intentionally stay English in every locale.

To regenerate the reports:

1. Open `demo/showcase_coco/showcase_coco.cvmlab.json` via **Open project**.
2. Switch between model runs in the workspace to review per-run metrics.
3. Click **Export Report** → choose HTML / CSV / XLSX / PDF and select the report language.
4. Open **Model Compare** and export pairwise or multi-model reports.

All report formats include AP metrics when available.
See `demo/showcase_coco/reports/README.md` for the full file list and validation notes.

## Why synthetic data?

Screenshots and demo content in public repositories must not contain real imagery
that may be proprietary or privacy-sensitive. CV Model Lab has been validated on
real internal object detection datasets, but those images and reports are not
published.
