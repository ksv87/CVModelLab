# Showcase COCO Dataset

[Русская версия](README.ru.md)
Synthetic road-scene dataset for CV Model Lab demos and screenshots.

> **No real images.** Every pixel is generated by
> `tools/generate_showcase_dataset/generate_showcase_dataset.py`
> using only geometric shapes and gradients. No photographs or copyrighted assets.

## Contents

| File / Directory | Description |
|-----------------|-------------|
| `images/` | 40 synthetic PNG scenes (960×540) |
| `annotations.json` | COCO ground truth — 5 categories, 87 annotations |
| `predictions_model_a.json` | Model A (Baseline) predictions — 72 boxes |
| `predictions_model_b.json` | Model B (High Recall) predictions — 106 boxes |
| `predictions_model_c.json` | Model C (High Precision) predictions — 70 boxes |
| `ap_metrics_model_a.json` | Precomputed COCO AP metrics for Model A |
| `ap_metrics_model_b.json` | Precomputed COCO AP metrics for Model B |
| `ap_metrics_model_c.json` | Precomputed COCO AP metrics for Model C |
| `reports/` | Generated EN/RU sample exports for full, pairwise, and multi-model reports (see reports/README.md) |

## Categories

| ID | Name | Description |
|----|------|-------------|
| 1 | `red_light` | Red traffic light signal |
| 2 | `yellow_light` | Yellow traffic light signal |
| 3 | `green_light` | Green traffic light signal |
| 4 | `pedestrian_sign` | Pedestrian crossing sign |
| 5 | `background_distractor` | Street lamp / circular sign (non-traffic-light) |

## Model behaviour

| Model | AP | AP50 | Recall | Characteristic |
|-------|----|------|--------|----------------|
| A — Baseline | 0.52 | 0.74 | ~64% | Balanced; some FP on distractors; misses 35% of small objects |
| B — High Recall | 0.47 | 0.70 | ~75% | Best recall; many FP on background and distractors |
| C — High Precision | 0.61 | 0.82 | ~67% | Best AP; high-confidence; misses 60–90% of small objects |

## Quick start in CV Model Lab

Click **Open project** on the project open screen and select
`demo/showcase_coco/showcase_coco.cvmlab.json`. The saved project loads all three
model runs and their precomputed AP metrics.

Alternatively, open files manually:

1. Open **annotations.json** as annotations
2. Add predictions: **predictions_model_a.json**, then add more runs with B and C
3. Import AP metrics per run via *Import AP metrics JSON* in the dashboard
